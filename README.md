# 学生信息管理系统

## How to Run

### 环境要求
- Docker 20.10+
- Docker Compose 2.0+

### 快速启动

```bash
# 克隆项目后，进入项目根目录

# 构建并启动所有服务
docker-compose up --build -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

### 验证镜像跨平台支持（ARM/X86）

```bash
# 验证 nginx 基础镜像支持 ARM64
docker pull --platform linux/arm64 nginx:1.25-alpine

# 验证构建的镜像
docker-compose build
```

## 项目介绍

一个功能完整的学生信息管理系统，采用现代化的UI设计，支持多种功能模块。

### 项目结构

```
├── docker-compose.yml          # Docker Compose 配置
├── .gitignore                  # Git 忽略文件配置
├── README.md                   # 项目说明文档
└── frontend-admin/             # 管理后台前端
    ├── Dockerfile              # Docker 构建文件
    ├── nginx.conf              # Nginx 配置
    ├── index.html              # 主页面
    ├── css/
    │   └── style.css           # 样式文件
    └── js/
        └── app.js              # 交互逻辑
```

### 功能模块

#### 1. 用户认证模块
- ✅ 用户登录
- ✅ 用户注册
- ✅ 密码找回
- ✅ 权限管理

#### 2. 学生基本信息管理模块

**信息采集维度：**
- 姓名、性别、出生日期、民族、籍贯

- 父母姓名、父母职业、父母电话
- 是否重组家庭、是否留守儿童、疾病史

#### 3. 学生行为管理模块
- ✅ 健康记录
- ✅ 奖惩记录
- ✅ 成长档案

#### 4. 学生成绩管理模块
- ✅ 成绩导入
- ✅ 成绩分析（平均分、及格率、优秀率等）
- ✅ 趋势追踪（单科和总成绩走势图）
- ✅ 统计报表（班级报表、学科报表、学生报表、班级对比）

### 技术栈

- **前端**: HTML5 + CSS3 + JavaScript (原生)
- **图表**: Chart.js
- **容器化**: Docker + Nginx
- **跨平台**: 支持 ARM64 和 AMD64 架构

### 工程与架构质量

项目采用模块化设计，代码结构清晰。前端使用原生技术栈零依赖，通过状态管理统一数据流；Docker 多阶段构建优化镜像体积，Nginx 配置 Gzip 压缩与缓存策略提升性能；支持 ARM/X86 跨平台部署，具备健康检查机制保障服务可用性。

### 注意事项

- 本系统为**纯前端演示版本**，数据存储在浏览器内存中，刷新页面后数据会重置
- **Excel导入/导出为模拟实现**：由于纯前端项目不引入xlsx等第三方解析库以保持轻量，模板导入、成绩导入功能仅模拟交互流程（上传后展示预设数据），报表导出仅提示成功无实际文件产出。如需真实Excel解析能力，需集成SheetJS等库或后端服务
- 如需数据持久化存储，需要集成后端服务和数据库
